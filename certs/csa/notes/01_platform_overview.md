# 分野1: Platform Overview and Navigation(配点 7%)

## 1. Now Platform の基本

- ServiceNow は **PaaS(Platform as a Service)** 型のクラウドプラットフォーム。単一のデータモデル上に ITSM / ITOM / HR / CSM などのアプリケーションが載る。
- 各顧客は専用の**インスタンス**(例: `company.service-now.com`)を持つ。**シングルテナント型**(インスタンスごとに独立したDB)。
- **リリース**: 年2回のメジャーリリース。都市名のアルファベット順で命名(例: Xanadu → Yokohama → Zurich)。
- **Now Support(旧HI)**: ServiceNowへのサポート依頼・インスタンス管理を行うポータル。

## 2. Next Experience UI(統合ナビゲーション)

現行のデフォルトUI。画面上部のヘッダーに主要メニューが集約されている。

| 要素 | 役割 |
|------|------|
| **All(すべて)** | 全アプリケーション・モジュールのメニュー(旧アプリケーションナビゲーター)。フィルター入力で絞り込み |
| **Favorites(お気に入り)** | ★を付けたモジュールへのショートカット。名前・アイコン編集可 |
| **History(履歴)** | 直近に開いたレコード・リスト・フォーム |
| **Workspaces** | ロールに応じたワークスペース(Agent Workspace等)一覧 |
| **Admin** | 管理者向けクイックアクセス(adminロール保持者のみ) |
| **グローバル検索** | ヘッダーの虫眼鏡。検索グループ・検索ソースで対象を制御 |
| **ユーザーメニュー** | 右上アバター。プロファイル・設定・impersonate・ログアウト |
| **Contextual App Pill** | 現在のアプリケーションスコープ・更新セットの表示/切替(開発者向け設定) |

- **モジュール**: メニューの1項目。リスト表示・フォーム表示・URL・スクリプト実行などのリンクタイプを持つ。
- **アプリケーションメニュー**: モジュールをまとめる親項目。ロールで表示制御できる。

## 3. Impersonation(ユーザー代理)

- adminは他ユーザーに**なりすまし(impersonate)**て、そのユーザーの見え方・権限を確認できる(`impersonator` ロールでも可)。
- impersonate中の操作は**そのユーザーの操作として記録**されるが、システムログにはimpersonateの事実も記録される。
- テスト目的でのアクセス権確認(ACL・UIポリシーの見え方)に必須のテクニック。

## 4. リストとフォーム(基本操作)

- **リスト**: テーブル内レコードの一覧。列ソート・フィルター・グループ化・インライン編集が可能。
  - **リストコントロールメニュー**(左上ハンバーガー)から列の設定・エクスポート等。
  - **ブレッドクラム**: 現在のフィルター条件を表示。クリックで条件を段階的に解除。条件は**左から右にAND**で絞り込まれる。
- **フォーム**: 1レコードの表示・編集画面。関連リスト(Related Lists)で子レコードを表示。
- **右クリックメニュー**(フォームヘッダー): Configure > Form Layout / Form Design、Export、履歴表示など管理操作の入口。

## 5. 検索

- **グローバル検索**: 複数テーブル横断。ワイルドカード(`*text`)や完全一致(`"text"`)が使える。
- **リスト検索**: 列ヘッダー上の検索行。`*`前方付与で部分一致。
- **Zing**: ServiceNowの検索エンジン(グローバル検索・ナレッジ検索等を支える)。

## 6. 設定(Preferences)

- ユーザーメニュー > Preferences でテーマ・表示密度・言語・タイムゾーン等を個人単位で設定。
- 管理者はシステムプロパティでインスタンス全体のデフォルトを設定(分野2参照)。

---

## 頻出ポイント

- All / Favorites / History の役割の区別
- モジュールとアプリケーションメニューの関係、ロールによる表示制御
- impersonate で何が確認できるか・誰ができるか
- ブレッドクラムの読み方(AND条件・クリックによる解除)
- リリースが年2回・都市名順であること

## 全体構造マップ

```
【Now Platform インスタンス】
  ├── シングルテナント(顧客ごとに独立したDB)
  ├── URL: company.service-now.com
  └── 年2回リリース(都市名順: Xanadu → Yokohama → Zurich ...)

         ┌─────────────────────────────────┐
         │      Next Experience UI (ヘッダー) │
         └───────────────┬─────────────────┘
                         │
          ┌──────────────┼──────────────┐
          ↓              ↓              ↓
        [All]       [Favorites]    [History]
          │           (★登録済み)   (直近閲覧)
          │
          ↓ フィルター入力で絞り込み
  ┌───────────────────────┐
  │  アプリケーションメニュー  │  ← ロールで表示制御
  └──────────┬────────────┘
             │
             ↓ クリック
  ┌───────────────────────┐
  │       モジュール        │  ← リンクタイプ: リスト/フォーム/URL/スクリプト
  └──────────┬────────────┘
             │
       ┌─────┴─────┐
       ↓           ↓
   【リスト】     【フォーム】
   (テーブル全体)  (1レコード)
       │               │
       │  ブレッドクラム   │  関連リスト
       │  [条件A > 条件B] │  (子レコード表示)
       │  ← AND で絞込   │
       ↓               ↓
   インライン編集     右クリックメニュー
   グループ化         └── Configure > Form Layout
   エクスポート            Form Design

【グローバル検索(Zing エンジン)】
  ヘッダー虫眼鏡
  └── 複数テーブル横断
       ├── *text  → 部分一致(前方ワイルドカード)
       └── "text" → 完全一致

【ユーザー管理 / 権限確認】
  sys_user テーブル(ユーザー情報)
      │
      ↓ adminが impersonate 実行
  対象ユーザーの目線でUI確認
      ├── ACL の効き方
      ├── UIポリシーの表示/非表示
      └── モジュールの見え方
  ※ 操作はそのユーザーとして記録
  ※ impersonate自体はシステムログに記録
```

## PDI 操作ガイド

PDI(Personal Developer Instance)は `developer.servicenow.com` で無料取得できる。

### タスク1: All メニューの構造を確認する

1. `All` をクリックし、左サイドパネルを開く。
2. フィルター入力欄に `user` と入力し、メニューが絞り込まれることを確認する。
3. `User Administration` (アプリケーションメニュー)の下に `Users`・`Groups`・`Roles` などのモジュールが並んでいることを確認する。
4. `Users` をクリックし、`sys_user` テーブルのリスト画面が開くことを確認する。
5. `Users` モジュールの右に表示される ★ をクリックして Favorites に登録し、ヘッダーの `Favorites` タブに表示されることを確認する。

### タスク2: リストのブレッドクラムを操作する

1. `All → Incident → All`(インシデント全件リスト)を開く。
2. 任意のレコードの `State` セル上で右クリック → `Filter By` を選択し、フィルターを追加する。
3. 画面上部のブレッドクラムに `State = New` のような条件が表示されることを確認する。
4. さらに `Priority` 列でも同様にフィルターを追加し、ブレッドクラムに2つ目の条件が AND で追加されることを確認する。
5. ブレッドクラムの最初の条件(左側)をクリックし、2つ目の条件が解除されることを確認する。

### タスク3: Impersonation を実行して権限の違いを確認する

1. ユーザーメニュー → `Impersonate User` をクリックする。
2. 検索ボックスに `Abel` と入力し、サンプルユーザー `Abel Tuter` を選択する。
3. 画面が切り替わり、ヘッダーに `Impersonating Abel Tuter` と表示されることを確認する。
4. `All` メニューを開き、admin 時と比べてメニュー項目が減っていることを確認する。
5. ユーザーメニュー → `End Impersonation` を選択し、元の admin セッションに戻る。

### タスク4: グローバル検索の動作を確認する

1. 虫眼鏡をクリックし、検索ボックスに `network` と入力して Enter を押す。
2. 検索結果ページが開き、複数のカテゴリに分かれて結果が表示されることを確認する。
3. 次に `*work` と入力して検索し、前方ワイルドカードで "network" などが引っかかることを確認する。
4. `"network outage"` (ダブルクォート付き)で検索し、完全一致フレーズのみがヒットすることを確認する。

## 試験との接続

「All / Favorites / History のどれを使うべきか」→ Allは探す、Favoritesは繰り返し使う、Historyは戻る。

「impersonate できるのは誰か、何のために使うか」→ admin または impersonator ロール保持者。ACL・ロールの効果をそのユーザーの目線で確認するため。

「HomepageとDashboardの違いは何か」→ Homepageは個人カスタマイズ可能なウィジェット画面。Dashboardはレポートを集めて共有するレポート集合画面。

「TagsとFavoritesの違いは何か」→ Favoritesはナビゲーション（モジュール・URL）のブックマーク。Tagsはレコードに付けるラベルで横断検索に使う。

「ブレッドクラムの条件をクリックするとどうなるか」→ クリックした条件より右側の条件が解除される。条件は左から右にANDで積み上がる。

「グローバル検索で部分一致検索するにはどうするか」→ `*` を前に付ける(例: `*work`)。`"..."` はフレーズ完全一致。ServiceNow固有の記法。
